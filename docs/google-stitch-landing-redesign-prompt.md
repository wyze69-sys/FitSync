# Google Stitch Prompt: FitSync Landing Page Redesign

Copy and paste this prompt into Google Stitch after uploading your current FitSync landing page files/assets.

---

Redesign and improve my existing FitSync landing page. Keep the current visual direction, but make it feel more accurate, complete, and connected to the real FitSync web app.

## Context

FitSync is a real fitness tracking web app, not a generic gym landing page. The product helps users track workouts, weight progress, nutrition targets, streaks, XP, levels, and long-term consistency.

FitSync has a protected dashboard after login, plus pages for:

- Dashboard
- Log Workout
- Nutrition
- Progress
- History
- Profile

## Current Landing Page

The current landing page already has a good premium fitness-tech look with:

- dark green / cream visual style
- animated dashboard-style graphics
- bento sections
- streak section
- CTA buttons

Do **not** completely replace the identity. Improve it so it feels more like the actual FitSync app and less like a generic fitness template.

## Main Goal

Make the landing page clearly explain what FitSync does and why users should register.

It should feel like a **fitness progress dashboard product**, not just a motivational gym website.

## Design Direction

Create a premium fitness-tech SaaS landing page.

The page should feel:

- serious
- modern
- dashboard-driven
- workout-first
- compact
- balanced
- useful
- slightly gamified, but not childish

Avoid:

- childish gaming style
- generic body transformation marketing
- fake influencer/gym stock-photo vibes
- exaggerated AI marketing
- random decorative sections that do not explain the product

Use:

- strong structure
- real product sections
- dashboard cards
- progress visuals
- XP/streak indicators
- nutrition target previews
- workout history previews

Keep the green/cream/dark fitness-tech palette, but make it more polished and consistent.

Use clean spacing, strong typography, and a responsive layout.

## Real FitSync Features to Show

### 1. Workout Logging

Users can log:

- Strength
- Cardio
- Yoga/Mobility
- Sports
- Plank/Hold workouts

Workout details can include:

- duration
- intensity
- sets/reps/weight when relevant
- notes

Calories and XP are calculated from real workout data.

### 2. Nutrition Planning

Users can see a daily calorie target.

The nutrition plan uses:

- profile data
- goal
- current weight
- target weight
- activity level
- recent workout calories

Show macro targets:

- protein
- carbs
- fat

Show food recommendations as grounded suggestions, not fake AI magic.

### 3. Progress Tracking

Users can track:

- weight
- BMI
- target weight progress
- trends

The dashboard shows:

- today’s workout summary
- calories
- XP
- level
- streak

### 4. Gamification

FitSync includes:

- XP
- levels
- streaks
- badges

Present gamification as motivation and consistency tracking, not childish game design.

Use serious labels such as:

- Consistency score
- Training streak
- Level progress
- XP earned from logged sessions

### 5. History

Users can review logged workouts by:

- date
- workout type
- duration
- calories
- XP

Show this as a clean timeline/table preview.

### 6. Profile-Based Personalization

The app uses profile data like:

- age
- gender
- height
- weight
- target weight
- activity level
- preferred workout type
- goal

Explain that better profile data creates more accurate calorie and nutrition targets.

## Copywriting Rules

Use plain, clear English.

Do not use fake numbers like:

- 10,000 users
- 99% success rate
- trusted by thousands

unless clearly shown as placeholder only.

Do not use exaggerated claims.

Avoid overusing “AI-powered”.

Avoid generic marketing clichés:

- Unleash your potential
- Next-gen fitness
- Revolutionize your journey
- Transform your life
- Elevate your fitness
- Seamless experience

Keep the tone:

- practical
- confident
- fitness-focused
- product-focused

## Visual Rules

Keep a palette based on:

- dark green
- cream
- white/off-white
- muted gold accent
- dark neutral text

Use:

- dashboard-style cards
- data visuals
- compact balanced card layouts
- product mockups
- progress indicators
- subtle motion

Avoid:

- huge empty sections
- too many giant cards
- generic equal 3-card feature rows
- childish icons
- emoji
- neon purple/blue gradients
- fake stock-photo gym energy

Use bento grids, staggered cards, product preview blocks, or asymmetric sections where useful.

Make the desktop layout use available width well.

Mobile must stack cleanly with no horizontal overflow.

Tap targets should be comfortable.

## Asset Instructions

Use the uploaded landing page assets if they fit, including:

- existing background images
- SVG illustrations
- Lottie animation
- dashboard visuals

But improve placement and section structure.

If an asset feels generic or unrelated, replace it with a more product-focused dashboard/mockup visual.

## Navigation

Landing page navigation should include:

- Home

- Progress
- Streaks
- Login
- Start Your Journey

## Important Scope

This is for the **public landing page only**.

Do not design the logged-in dashboard pages.

The landing page should lead users to register or login.

Create a redesigned landing page that still feels like my current page, but is:

- more complete
- more accurate to FitSync
- more product-focused
- more professional
- more connected to the real app features
